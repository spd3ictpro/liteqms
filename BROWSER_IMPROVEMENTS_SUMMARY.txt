# LiteQMS — Browser Display Improvements Summary

## Goal
Improve the existing /Display page experience for TV use — no new app needed.

---

## 5 Improvements

### 1. Reconnection UI
Problem: Currently when SignalR disconnects, the display just freezes with no indication.
Solution: Adds a visible banner ("Reconnecting..." / "Offline") + status dot in header.
- Yellow banner = reconnecting
- Red banner = offline
- Green dot = connected
- Yellow dot = reconnecting
- Red dot = offline
- Auto-hides when connection restored

### 2. Wake Lock API
Problem: TV screen may auto-dim or sleep after inactivity.
Solution: Keeps the screen awake while the display tab is open.
- Uses navigator.wakeLock.request('screen')
- Requests on first user click (audio enable click)
- Re-acquires when tab becomes visible again
- Graceful fallback if API not supported

### 3. Fullscreen Button
Problem: Browser address bar and tabs take up screen space.
Solution: One-click toggle to true fullscreen, hides automatically after 3s.
- Small button in header bar (top-right corner)
- Toggles document.documentElement.requestFullscreen() / document.exitFullscreen()
- Icon changes between expand/collapse
- Auto-hides after 3 seconds of inactivity
- Reappears on mouse move

### 4. Better Audio Overlay
Problem: Current overlay is basic.
Solution: Cleaner design, clearer instructions, smooth fade-out, only shows once.
- Prominent "Click anywhere to enable audio" message
- Smooth fade-out animation after enabled
- Never returns after being enabled once

### 5. Burn-in Prevention
Problem: Static patient number displayed for hours can burn into LED panels.
Solution: Subtly shifts content 1-2px every 5 minutes — imperceptible but protective.
- Uses CSS transform: translate() on the container
- Randomizes direction each cycle
- Runs every 5 minutes

---

## Files to Change

- Pages/Display.cshtml — New UI elements (fullscreen button, reconnection banner)
- wwwroot/css/display.css — New styles for banner, status dot, fullscreen button
