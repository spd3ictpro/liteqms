ITEM 2 — Input Validation Hardening
============================================================

Goal: Prevent malformed input, XSS attempts, and anti-forgery token errors.

Changes:

2a. Pages/Index.cshtml.cs
    - Added RoomNumber.Trim() before validation
    - Added character whitelist: only letters, digits, spaces, hyphens, slashes
    - Added max length check (50 chars)
    - Rejects input like "<script>", "../../", garbage with clear error message

2b. Hubs/QueueHub.cs
    - Added same room number validation in CallPatient hub method
    - Protects against rogue SignalR clients

2c. Pages/History.cshtml.cs
    - Added date range clamping: if from > to, clamp from to to
    - Capped range to max 90 days to prevent huge DB scans
    - Combined into single WHERE clause

2d. Moved CNA toggle from form POST to SignalR invoke
    - Before: CNA forms were generated as raw HTML in JS, POSTed to /CallPanel?handler=CNA
    - Problem: Dynamic forms lacked anti-forgery token → would get 400 errors in production
    - Fix: CNA now works through connection.invoke("ToggleCNA", id) — same as recall

    Files changed:
    2d-i. Hubs/QueueHub.cs — added ToggleCNA(int callRecordId) hub method
    2d-ii. Pages/CallPanel.cshtml.cs — removed OnPostCNA handler entirely
    2d-iii. Pages/CallPanel.cshtml — replaced CNA <form> with plain <button data-cna-id="...">

        - Global click handler: added .cna-btn case calling connection.invoke("ToggleCNA", id)
